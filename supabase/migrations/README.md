# supabase/migrations

Todas las migraciones SQL de Prisma se generan aquí.

**No editar manualmente.** Usar los scripts desde la raíz:

```bash
npm run db:migrate          # crear nueva migración (dev)
npm run db:migrate:deploy   # aplicar migraciones en producción
npm run db:migrate:reset    # resetear la base de datos (dev)
npm run db:push             # sync schema sin migración
npm run db:studio           # abrir Prisma Studio
npm run db:generate         # regenerar el cliente Prisma
```
